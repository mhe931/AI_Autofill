AI Form Filler - Documentation

An intelligent automation suite developed by Daniel Ebrahimzadeh.

How It Works: The extension combines your professional profile (Daniel Ebrahimzadeh, 14+ years Data Engineering, M.Sc. at University of Vaasa) with the target form HTML into a single LLM-ready prompt. You paste it into any AI, copy the JSON answer back, and the extension fills the form.

3-Step Workflow

  1. 1 → Extract (Prompt + HTML)
    Navigate to the target form page. Click "Extract Form HTML". The extension scrapes all form elements, merges them with your applicant profile and historical data, and produces a complete LLM prompt. This prompt appears in the textarea and is copied to your clipboard.
  2. 2 → Paste to AI
    Open an LLM (Gemini, ChatGPT, Claude, etc.). Click "Paste to LLM". The extension reads directly from its textarea and injects the prompt into the AI chat input. No clipboard dependency at this step.
  3. 3 → Paste AI JSON & Execute
    Copy the LLM JSON response. Paste it into the extension textarea (replacing the prompt). Click "Execute Injection". The extension strips any ```json markdown fences, validates JSON, then maps keys to form fields using multi-signal matching (id, name, label, placeholder, container text).

Data Warehouse & Persistence

All data stays local via chrome.storage.local:

  • Capture Current Form Data: Scrapes all filled fields into timestamped records for future prompt augmentation.
  • View Data Table: Opens a formatted HTML table of all captured context.
  • Export / Import: Full JSON backup and restore for cross-machine portability.

Troubleshooting

Clipboard Permission Errors

If extraction fails to copy to clipboard:

  1. Click the Lock Icon in the URL bar.
  2. Select Site Settings.
  3. Set Clipboard to Allow.
  4. Reload the page and retry.

Note: Even if clipboard copy fails, the prompt still loads into the textarea. You can copy it manually.

Connection Errors

If you see "Receiving end does not exist" or "Reload the target page to reconnect":

  1. This means the extension listener on the target page has been disconnected (e.g., after a browser update or long idle period).
  2. Refresh the target form page (F5 or Ctrl+R) to restart the extension listener.
  3. Try the action again. The extension will also attempt automatic re-injection on the first failure.

URL Support Limitation

The extension cannot inject into chrome:// or edge:// pages. Navigate to an http:// or https:// target.

Best Practices

  • Active Tab Focus: Click into the target tab before pressing Extract.
  • LLM Input Focus: Click inside the LLM chat box before pressing Paste.
  • JSON Fences: The extension auto-strips ```json wrappers. If injection still fails, verify JSON structure manually.
  • Capture Often: Run Capture after each form to build historical context for future prompts.
