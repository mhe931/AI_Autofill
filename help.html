AI Form Filler - Documentation

A general-purpose toolkit developed by Daniel Ebrahimzadeh.

Overview: This extension empowers users to seamlessly transfer full form contexts to an external LLM, augment prompts with historical local data, and dynamically inject mapped responses back into the DOM using intelligent fuzzy matching algorithms.

3-Button Analyze-Generate-Inject Workflow

  1. Step 1: HTML Extraction. Open your target form. Click "Extract Form HTML" in the popup. This automatically scrapes the parent form boundaries (or the full DOM if nested) and injects a standardized LLM system instruction prefix into your clipboard.
  2. Step 2: AI Context Processing. Open an external generative AI (like Gemini, ChatGPT, or Claude). Click the "Paste to LLM" button to deploy the aggregated HTML string along with historical contextual tabular data into the chat field.
  3. Step 3: Dynamic Form Injection. After the LLM constructs your mapped JSON schema, copy its output. Go back to the form and click "Execute Injection" to dynamically match those generated values to all relevant input fields and textareas.

Data Warehouse & Persistence

We advocate a strictly "Local First" architecture:

  • Capture Current Form Data: After finishing a form logically, press Capture to safely register those text responses or exact local file paths into a persistent database.
  • View Data Table: Generate an immediate HTML visualization of all saved context mappings.
  • Export / Import: Use JSON blobs for system-agnostic migrations across workstations.
