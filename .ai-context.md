# AI Form Filler - Context

**Project Name**: AI Form Filler

**Current Architecture**: 
- Manifest V3
- Event-driven injection ( Zero Framing logic kept intact )

**Primary Dependencies**: 
- Vanilla JavaScript (No external framework overhead)

**Current Goal**: 
- Shift toward dynamic JSON injection [cite: 2026-03-12]. Replace hardcoded data with dynamic mapping that reads from user-provided JSON. Iterate through keys directly for universal form filling across different domains.
- Update to a decoupled LLM-assisted workflow for dynamic form mapping and injection. This involves an automated "Analyze-Generate-Inject" pipeline via UI buttons allowing for HTML extraction directly to clipboard, direct LLM input pasting, and logic iteration across `document.forms` with fuzzy matching implementation [cite: 85, 95].
- Implement "Local First" Data Warehouse architecture [cite: 2026-03-12]. A relational tabular schema saves mapping associations (`Field Label`, `Value`, `Source URL`, `Timestamp`) including mapping system file paths. The generated LLM prompt dynamically augments using context injection drawn directly from the local table. Features complete backup/restore tooling.
